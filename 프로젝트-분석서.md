## 2. MVP 범위 정의

### MVP 포함 (Phase 0~3, 약 6~8주)

| 기능 | 상세 |
|------|------|
| 랜딩 페이지 | 브랜드 소개 + 멍BTI/냥BTI 선택 |

| 모바일 반응형 | 모바일 퍼스트 UI |

- Vision AI (사진/영상 분석)
- 모순 감지 → 추가 질문
- 신뢰도 점수 표시
- 보호자 MBTI 궁합

## 6. AI 파이프라인

### 하이브리드 점수 산출 (핵심 설계 결정)

```
[객관식 응답] → 규칙 기반 가중치 합산 → 4축 원시 점수 (-1~+1 정규화)
                                              ↓
[자유 입력 있을 때만] → LLM Call 1: 점수 보정 (±0.2 범위)
                                              ↓
                              최종 4축 점수 → 유형 코드 결정
                                              ↓
                              LLM Call 2: 맞춤 결과 텍스트 생성
```

**유형 코드 결정 (결정적):**
- energy ≥ 0 → A, < 0 → C
- social ≥ 0 → S, < 0 → M
- sensitivity ≥ 0 → D, < 0 → B
- curiosity ≥ 0 → E, < 0 → H

**LLM 호출: 최대 2회/진단**
- Call 1: 자유 입력 분석 + 점수 보정 (자유 입력 없으면 스킵)
- Call 2: 유형+응답 기반 맞춤 성격 설명 + 궁합 텍스트 생성
- 예상 비용: 월 10,000건 기준 ~$8

**Fallback:** LLM 장애 시 규칙 기반 점수 + pet_types.base_description 템플릿으로 결과 제공

## 7. 사용자 플로우 & UI 설계

### 화면별 렌더링 전략

| 페이지 | 렌더링 | SEO |
|--------|--------|-----|
| `/` 랜딩 | SSG | O |
| `/select` 파트 선택 | SSG | O |
| `/test/[type]/*` 테스트 | CSR | X |
| `/result/[id]` 결과 | SSR | O (공유 진입점) |

### 핵심 UI 포인트

1. **랜딩**: 꼬리 흔드는 애니메이션 + CTA (above the fold) + 참여자 카운터
2. **파트 선택**: 강아지/고양이 카드 2장, 탭하면 scale-up + 자동 이동
3. **질문**: Swiper.js 스와이프, 선택 시 0.4초 후 자동 다음 이동, 프로그레스바
4. **로딩**: 꼬리 흔들림 애니메이션 + 단계별 체크 연출 (최소 5초)
5. **결과**: 캐릭터 일러스트 + AI 맞춤 설명 + 궁합 + 공유 버튼 3종

### 공유 구현

- **카카오톡**: Kakao SDK `Share.sendDefault` (피드 타입)
- **링크 복사**: `navigator.clipboard.writeText`
- **OG 이미지**: Next.js `ImageResponse`로 동적 생성 (1200x630)

## 12. PM 핵심 조언

1. **프롬프트 엔지니어링이 핵심 기술 자산** — Phase 0에서 충분히 테스트하고 안정화
2. **질문 수 12개로 축소 강력 추천** — 모바일에서 15개는 이탈률 급증
3. **결과 카드의 "공유 욕구"가 성패** — 기술보다 결과 텍스트의 재미/공감에 투자
4. **LLM 의존도 최소화** — 규칙 기반 점수 + LLM 보정 하이브리드로 안정성 확보
5. **빠른 런칭이 핵심** — 완벽보다 속도, 2차에서 고도화
